\section{Introducción}

Este proyecto presenta el desarrollo de dos modelos de clasificación supervisada
usando técnicas de inteligencia artificial implementadas en PyTorch. El trabajo
abarca dos dominios distintos del problema agroindustrial:

\begin{itemize}
  \item \textbf{Componente tabular:} clasificación binaria sobre un dataset
        de variables numéricas, usando un Perceptrón Multicapa (MLP).
  \item \textbf{Componente de imágenes:} clasificación multiclase de hojas
        de papa a partir de fotografías, usando una Red Neuronal Convolucional (CNN).
\end{itemize}

En ambos casos se siguió un flujo completo: exploración del dataset,
preprocesamiento, diseño del modelo, entrenamiento, evaluación con métricas
y análisis crítico de los resultados.

\subsection{Preprocesamiento}

El pipeline implementado en \texttt{preprocess.py} realiza los siguientes pasos:

\begin{enumerate}
  \item Carga del archivo ARFF con \texttt{scipy.io.arff} y conversión a DataFrame.
  \item Limpieza de la variable objetivo: decodificación de \texttt{bytes} a \texttt{string}.
  \item Separación de $X = [V1, V2, V3, V4, V5]$ e $y$.
  \item Codificación con \texttt{LabelEncoder}: \texttt{'1'} $\to$ 0, \texttt{'2'} $\to$ 1.
  \item División 80\,\%/20\,\% con muestreo estratificado.
  \item Escalado con \texttt{StandardScaler} ajustado solo sobre train:
        $x_{\text{norm}} = (x - \mu_{\text{train}}) / \sigma_{\text{train}}$
  \item Conversión a tensores PyTorch.
\end{enumerate}

\begin{lstlisting}[language=Python, caption={Pipeline de preprocesamiento (\texttt{preprocess.py})}]
def preprocess_pipeline(filepath, feature_cols=None,
                        target_col="Class", test_size=0.2,
                        random_state=42):
    if feature_cols is None:
        feature_cols = ["V1", "V2", "V3", "V4", "V5"]

    df = load_arff(filepath)
    df = clean_target(df, target_col)
    X, y = split_features_target(df, feature_cols, target_col)
    y, le = encode_target(y)
    X_train, X_test, y_train, y_test = split_data(
        X, y, test_size, random_state)
    X_train, X_test, scaler = scale_features(X_train, X_test)
    X_train_t, X_test_t, y_train_t, y_test_t = to_tensors(
        X_train, X_test, y_train, y_test)
    return {"X_train": X_train_t, "X_test": X_test_t,
            "y_train": y_train_t, "y_test": y_test_t,
            "num_classes": len(le.classes_)}
\end{lstlisting}

\subsection{Modelo: Perceptrón Multicapa (MLP)}

\subsubsection{Fundamento matemático}

Un MLP transforma la entrada $\mathbf{x} \in \mathbb{R}^5$ mediante capas
sucesivas de transformaciones lineales y no lineales. Cada capa realiza:

\[
  \mathbf{z}^{(l)} = \mathbf{W}^{(l)} \mathbf{a}^{(l-1)} + \mathbf{b}^{(l)}
  \qquad
  \mathbf{a}^{(l)} = \text{ReLU}(\mathbf{z}^{(l)}) = \max(0,\, \mathbf{z}^{(l)})
\]

La capa de salida produce logits $\mathbf{z}^{(\text{out})} \in \mathbb{R}^2$
sin activación, ya que \texttt{CrossEntropyLoss} incorpora \texttt{LogSoftmax}
internamente.

\subsubsection{Arquitectura}

    2 & \texttt{Linear} + ReLU  & 16 & 8  \\
    3 & \texttt{Linear} (salida) & 8 & 2  \\
    \bottomrule
  \end{tabular}

\begin{lstlisting}[language=Python, caption={Definición del MLP (\texttt{model.py})}]
class MLP(nn.Module):
    def __init__(self, input_dim: int, num_classes: int):
        super().__init__()
        self.fc1   = nn.Linear(input_dim, 16)
        self.relu1 = nn.ReLU()
        self.fc2   = nn.Linear(16, 8)
        self.relu2 = nn.ReLU()
        self.fc3   = nn.Linear(8, num_classes)

    def forward(self, x):
        x = self.relu1(self.fc1(x))
        x = self.relu2(self.fc2(x))
        return self.fc3(x)   # logits crudos, sin Softmax
\end{lstlisting}

\subsection{Entrenamiento}

\subsubsection{Manejo del desbalance de clases}

Dado el desbalance severo (ratio $\approx$17:1), se aplicaron pesos de clase
con la fórmula estándar equivalente a \texttt{class\_weight='balanced'}
de scikit-learn:

\[
  w_i = \frac{n_{\text{total}}}{n_{\text{clases}} \times n_i}
\]

Los pesos resultantes con el dataset fueron:
\begin{itemize}
  \item Clase 0 (3.662 muestras): $w_0 = 0{,}5285$
  \item Clase 1 (209 muestras):   $w_1 = 9{,}2608$
\end{itemize}

\begin{lstlisting}[language=Python, caption={Cálculo de pesos de clase (\texttt{train.py})}]
def compute_class_weights(y_train):
    class_counts = torch.bincount(y_train)
    n_total      = len(y_train)
    n_classes    = len(class_counts)
    weights = n_total / (n_classes * class_counts.float())
    return weights

weights   = compute_class_weights(y_train)
criterion = nn.CrossEntropyLoss(weight=weights)
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)
\end{lstlisting}

\subsection{Preprocesamiento}

Las imágenes se cargan con \texttt{torchvision.datasets.ImageFolder}, que
detecta las clases automáticamente desde la estructura de carpetas. El pipeline
de transformaciones es:

\begin{enumerate}
  \item \textbf{Resize} a $128 \times 128$ px para unificar dimensiones.
  \item \textbf{ToTensor}: $(H \times W \times C)$ uint8 $[0,255]$
        $\to$ $(C \times H \times W)$ float32 $[0,1]$.
  \item \textbf{Normalize} canal a canal:
        $x_{\text{norm}} = (x - 0.5) / 0.5 \in [-1, 1]$
\end{enumerate}

\begin{lstlisting}[language=Python, caption={Transformaciones e ImageFolder (\texttt{dataset.py})}]
transform = transforms.Compose([
    transforms.Resize((128, 128)),
    transforms.ToTensor(),
    transforms.Normalize(mean=[0.5, 0.5, 0.5],
                         std =[0.5, 0.5, 0.5]),
])
dataset = datasets.ImageFolder(root=data_path, transform=transform)
train_loader = DataLoader(train_dataset, batch_size=32, shuffle=True)
test_loader  = DataLoader(test_dataset,  batch_size=32, shuffle=False)
\end{lstlisting}

\subsection{Modelo: Red Neuronal Convolucional (CNN)}

\subsubsection{Fundamento matemático}

Una CNN extrae características espaciales mediante filtros aprendibles.
La operación de convolución 2D es:

\[
  S[i,j] = \sum_{m}\sum_{n} I[i+m,\, j+n] \cdot K[m,n]
\]

donde $I$ es el mapa de entrada y $K$ el filtro (kernel). El \textbf{MaxPooling}
reduce la resolución tomando el máximo de cada ventana $2 \times 2$, logrando
invariancia a traslaciones y reduciendo parámetros.

\subsubsection{Arquitectura y flujo de dimensiones}

\begin{table}[H]
  \centering
  \small
  \begin{tabular}{llcc}
    \toprule
    \textbf{Capa} & \textbf{Tipo} & \textbf{Entrada} & \textbf{Salida} \\
    \midrule
    Conv1 + ReLU & \texttt{Conv2d(3,16,k=3,p=1)}  & $(B,3,128,128)$  & $(B,16,128,128)$ \\
    MaxPool1     & \texttt{MaxPool2d(2,2)}          & $(B,16,128,128)$ & $(B,16,64,64)$   \\
    Conv2 + ReLU & \texttt{Conv2d(16,32,k=3,p=1)} & $(B,16,64,64)$   & $(B,32,64,64)$   \\
    MaxPool2     & \texttt{MaxPool2d(2,2)}          & $(B,32,64,64)$   & $(B,32,32,32)$   \\
    Flatten      & —                                & $(B,32,32,32)$   & $(B,32768)$      \\
    FC1 + ReLU   & \texttt{Linear(32768,64)}        & $(B,32768)$      & $(B,64)$         \\
    FC2 (salida) & \texttt{Linear(64,3)}            & $(B,64)$         & $(B,3)$          \\
    \bottomrule
  \end{tabular}
  \caption{Arquitectura CNN y flujo de dimensiones ($B$ = tamaño de lote).}
\end{table}

\begin{lstlisting}[language=Python, caption={Definición de la CNN (\texttt{model.py})}]
class CNN(nn.Module):
    def __init__(self, num_classes: int):
        super().__init__()
        self.conv1   = nn.Conv2d(3,  16, kernel_size=3, padding=1)
        self.relu1   = nn.ReLU()
        self.pool1   = nn.MaxPool2d(2, 2)
        self.conv2   = nn.Conv2d(16, 32, kernel_size=3, padding=1)
        self.relu2   = nn.ReLU()
        self.pool2   = nn.MaxPool2d(2, 2)
        self.flatten = nn.Flatten()
        self.fc1     = nn.Linear(32 * 32 * 32, 64)
        self.relu3   = nn.ReLU()
        self.fc2     = nn.Linear(64, num_classes)

    def forward(self, x):
        x = self.pool1(self.relu1(self.conv1(x)))
        x = self.pool2(self.relu2(self.conv2(x)))
        x = self.flatten(x)
        x = self.relu3(self.fc1(x))
        return self.fc2(x)   # logits crudos, sin Softmax
\end{lstlisting}

\subsection{Entrenamiento}

El modelo se entrenó durante \textbf{10 épocas} iterando sobre batches de 32
imágenes. Se usó \texttt{CrossEntropyLoss} sin pesos (clases balanceadas) y
\texttt{Adam} con $\alpha = 0{,}001$.

\begin{lstlisting}[language=Python, caption={Bucle de entrenamiento por batches (\texttt{train.py})}]
for epoch in range(1, epochs + 1):
    running_loss = 0.0
    for images, labels in train_loader:
        images, labels = images.to(device), labels.to(device)
        logits = model(images)
        loss   = criterion(logits, labels)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
        running_loss += loss.item()
    avg_loss = running_loss / len(train_loader)
    print(f"  Epoca {epoch}/{epochs}  |  Loss: {avg_loss:.4f}")
\end{lstlisting}

\subsection{Análisis}

La caída de la pérdida de 0.7736 a 0.0226 en solo 10 épocas indica que la CNN
aprende características discriminativas de forma efectiva. La accuracy del
\textbf{92.39\,\%} es un resultado sólido considerando el tamaño reducido del
dataset (456 imágenes).

El leve repunte en época~3 (0.4436 tras 0.3956) es normal en entrenamiento
por batches con \texttt{shuffle=True}. A partir de época~4 la tendencia es
consistentemente decreciente.

La pérdida muy baja al final (0.0226) sugiere posible \textbf{sobreajuste}.
Para producción se recomendaría aplicar \textit{data augmentation} (rotaciones,
volteos, cambios de brillo) y \textit{Dropout} como regularización.

\section{Conclusiones}

Este proyecto demostró el desarrollo completo de dos pipelines de clasificación
supervisada en PyTorch aplicados a problemas agroindustriales reales.

\textbf{Componente tabular:}
\begin{itemize}
  \item El desbalance severo (ratio 17:1) no puede ignorarse: sin manejo, el
        modelo produce accuracy ficticia del 94.6\,\% con 0\,\% de aciertos
        en la clase minoritaria.
  \item Los pesos de clase con $w_i = n_{\text{total}}/(n_{\text{clases}} \times n_i)$
        logran tasas equilibradas de 67.9\,\% y 69.2\,\% por clase.
  \item La convergencia lenta sugiere que el problema requiere más datos o
        features más discriminativas.
\end{itemize}

\textbf{Componente de imágenes:}
\begin{itemize}
  \item La CNN alcanza 92.39\,\% de accuracy en 10 épocas, demostrando la
        efectividad de las arquitecturas convolucionales para clasificación
        de imágenes agrícolas.
  \item La pérdida baja final indica posible sobreajuste; en producción se
        recomendaría data augmentation y regularización.
\end{itemize}

\textbf{Reflexión:} La elección correcta de la función de pérdida, el manejo
del desbalance y el preprocesamiento adecuado son tan importantes como la
arquitectura. La comprensión del problema precede a la solución técnica.
